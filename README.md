### 📌 Overview
This repository documents a full progression of creative coding work, moving from fundamental drawing and interaction concepts to advanced, installation‑ready generative systems.
Across this body of work, sketches evolve from simple static compositions into responsive worlds driven by mathematics, noise, physics, sensors, sound, vision, and real‑time user interaction.
The project demonstrates not only technical proficiency, but a clear conceptual trajectory:

From drawing images → to building behaviors → to designing living systems


### 🧠 Core Themes
The collection explores several recurring themes:
- Generative form and repetition
- Motion, noise, and organic variation
- Interaction and state management
- Data as a creative material
- Simulation and autonomy
- Architectural, spatial, and environmental thinking
- Tool‑building for projection and installation
- Responsive systems driven by real‑world input


### 🗂 Project Structure
Each sketch is self‑contained and documented individually, but conceptually grouped by difficulty and topic.

### 📘 Conceptual Progression

#### :one: ️Foundations: Form, Composition, and Repetition
Early sketches establish basic visual language and structure:
- Static compositions
- Shape placement and contrast
- Loops and repetition
- Simple random systems

Key concepts: coordinate systems, repetition, visual balance


#### :two: Interaction & State
User input and real‑time change are introduced:
- Mouse‑driven visuals
- Conditional logic
- State switching
- Accumulative drawing

Key concepts: interaction, setup() vs draw(), program state


#### :three: Motion, Noise & Time
Movement becomes smoother and more organic:
- Linear motion and boundaries
- Perlin noise for natural variation
- Trigonometric paths
- Time‑based systems

Key concepts: noise, sine/cosine, smooth animation


#### :four: Abstraction & Systems Thinking
Code begins to scale:
- Custom functions
- Nested loops
- Grid systems
- Reusable visual logic

Key concepts: abstraction, modularity, code reuse


#### :five: Objects, Particles & Physics
Behavior becomes object‑based:
- Classes and lists
- Particle systems
- Vector‑based motion
- Steering behaviors

Key concepts: object‑oriented programming, physics modeling


#### :six: Data, Images & Pixels
The screen is treated as data:
- Image sampling
- Pixel array manipulation
- Procedural raster effects

Key concepts: low‑level graphics, data‑driven visuals


#### :seven: Recursion & Emergence
Complex form emerges from simple rules:
- Recursive fractals
- Self‑similar systems
- Hierarchical transformations

Key concepts: recursion, fractals, emergent complexity


#### :eight: 3D Worlds & Navigation
Flat sketches become spatial environments:
- Procedural 3D terrain
- Camera systems and follow cams
- World navigation
- Terrain‑aware agents

Key concepts: 3D space, camera logic, world consistency


#### :nine: Tools for Projection & Installation
The focus shifts from artworks to utilities:
- Manual keystone calibration
- Architectural masking systems
- Input mapping and control design

Key concepts: tool‑building, installation workflows


#### :keycap_ten: Project Systems: Responsive Environments
The final works integrate multiple systems simultaneously:
- Acoustic Brutalism: sound‑driven geometry
- Dynamic Terrain Navigation: agents interacting with procedural worlds
- Autonomous Agents: behavior‑based motion and intention
- Bioluminescent Ivy System: a living, sensor‑driven architectural organism responding to sound, motion, and light

Key concepts: multi‑input systems, adaptive behavior, installation‑ready design


#### 🛠 Technologies & Methods
- Processing / py5 (Python Mode)
- Perlin noise & procedural generation
- Vector mathematics & physics
- Object‑oriented programming
- Audio input (microphone analysis)
- Computer vision (camera brightness & motion)
- Pixel‑level manipulation
- 3D rendering (P3D)
- Projection‑mapping workflows
